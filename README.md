# 🏠 CMRU Dormitory Management System

ระบบบริหารจัดการหอพักออนไลน์ สำหรับมหาวิทยาลัยราชภัฏเชียงใหม่ 
พัฒนาระบบเพื่อให้ผู้ดูแลหอพักและผู้เช่าสามารถจัดการสิ่งต่างๆ ได้ง่ายขึ้นผ่านเว็บแอปพลิเคชัน

---

## 🚀 โปรเจคนี้ทำอะไรได้บ้าง? (Features)

โปรเจคนี้แบ่งการทำงานหลักๆ ออกเป็น 2 ส่วน คือ **ผู้ดูแลระบบ (Admin)** และ **ผู้เช่า (Tenant)**:

### 👑 ฝั่งผู้ดูแลระบบ (Admin)
- 📊 **Dashboard:** ดูภาพรวมสถิติหอพัก สรุปห้องว่าง/ห้องเต็ม
- 🏠 **จัดการห้องพัก:** เพิ่ม แก้ไข ลบ ข้อมูลห้องพัก พร้อมอัปโหลดรูปภาพได้
- 👥 **จัดการผู้เช่า:** ดูข้อมูลผู้เช่าและประวัติการเช่า
- 📋 **จัดการการจอง:** อนุมัติหรือปฏิเสธคำขอจองห้องพัก
- 🔧 **แจ้งซ่อม:** รับเรื่องและอัปเดตสถานะการแจ้งซ่อมจากผู้เช่า
- 💰 **จัดการบิล:** สร้างบิลค่าเช่า/ค่าน้ำ/ค่าไฟ และตรวจสอบสลิปการโอนเงิน
- 💬 **แชท:** คุยกับผู้เช่าได้แบบเรียลไทม์
- ⚙️ **ตั้งค่าเว็บไซต์:** ปรับเปลี่ยนรูปภาพแบนเนอร์ ข้อมูลติดต่อของหอพัก 

### 👤 ฝั่งผู้เช่า (Tenant)
- 🏠 **จองห้องพัก:** ดูห้องที่ว่าง และกดส่งคำขอจองห้อง
- 💳 **ชำระเงิน:** ดูบิลรายเดือน และอัปโหลดสลิปเพื่อแจ้งชำระเงิน
- 🔧 **แจ้งซ่อม:** ส่งเรื่องแจ้งซ่อมพร้อมแนบรูปภาพได้
- 💬 **แชท:** สอบถามข้อมูลหรือคุยกับแอดมินโดยตรง
- 🔔 **แจ้งเตือน:** มีระบบ Push Notification แจ้งเตือนสถานะต่างๆ

## 🛠️ โปรเจคนี้ใช้อะไรบ้าง? (Tech Stack)

โปรเจคนี้พัฒนาด้วยเทคโนโลยีสมัยใหม่ เพื่อให้เว็บเร็วและใช้งานง่าย:

- **Frontend & Framework:** 
  - `Next.js (v16)` - React framework สำหรับสร้างเว็บ
  - `React (v19)` - UI Library
  - `Tailwind CSS (v4)` - จัดการเรื่องความสวยงาม (Styling)
  - `TypeScript` - เพื่อเขียนโค้ดให้ปลอดภัยและลดบั๊ก
- **Backend & Database:**
  - `Firebase` - ใช้ครบวงจรทั้ง ระบบล็อกอิน (Auth), ฐานข้อมูล (Firestore) และแจ้งเตือน (Cloud Messaging)
  - `Firebase Admin SDK` - ใช้จัดการระบบหลังบ้านในฝั่ง Server
- **Storage (เก็บรูปภาพ):**
  - `@vercel/blob` - สำหรับเก็บไฟล์รูปภาพต่างๆ (เช่น รูปห้อง, สลิปโอนเงิน)
- **เครื่องมือเสริมอื่นๆ:**
  - `lucide-react` - ไอคอนสวยๆ 
  - `react-easy-crop` - ระบบตัดภาพและซูมภาพก่อนอัปโหลด
  - `@react-pdf/renderer` - สร้างไฟล์ PDF (เช่น ใบเสร็จ)
- **Package Manager:**
  - `Bun` - ตัวจัดการแพ็กเกจที่รวดเร็ว (แทน npm/yarn)

## 💻 วิธีเตรียมความพร้อมเมื่อเอาไปรันที่เครื่องตัวเอง (Setup Guide)

สำหรับเพื่อนๆ หรือนักพัฒนาที่ต้องการนำโปรเจคนี้ไปรันที่เครื่องตัวเอง ให้ทำตามขั้นตอนนี้:

### 1. สิ่งที่ต้องมีในเครื่อง (Prerequisites)
- [Node.js](https://nodejs.org/) (เวอร์ชัน 18 ขึ้นไป)
- [Bun](https://bun.sh/) (แนะนำให้ลงไว้ เพราะโปรเจคนี้ใช้ Bun ในการจัดการแพ็กเกจ รันคำสั่งรันสคริปต์)

### 2. โคลนโปรเจคและติดตั้งแพ็กเกจ
เปิด Terminal หรือ Command Prompt แล้วรันคำสั่ง:
```bash
# Clone โปรเจคลงเครื่อง
git clone <URL_ของ_Repository>

# เข้าไปในโฟลเดอร์โปรเจค
cd CMRU_Dormitory_System

# ติดตั้ง Dependencies ทั้งหมด
bun install
```

### 3. ตั้งค่า Environment Variables (.env)
โปรเจคนี้มีการเชื่อมต่อกับ Firebase และ Vercel Blob ดังนั้น **จำเป็นต้องมีคีย์ลับ** เพื่อให้ระบบทำงานได้ 

1. สร้างไฟล์ชื่อ `.env.local` ไว้ที่โฟลเดอร์นอกสุดของโปรเจค (Root directory)
2. นำค่าเหล่านี้ไปใส่ในไฟล์ `.env.local` (สามารถขอค่าเหล่านี้จากเพื่อนที่เป็นเจ้าของโปรเจค):

```env
# Firebase Client SDK (ตั้งค่าเชื่อมต่อ Firebase ฝั่งหน้าเว็บ)
NEXT_PUBLIC_FIREBASE_API_KEY=
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=
NEXT_PUBLIC_FIREBASE_PROJECT_ID=
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=
NEXT_PUBLIC_FIREBASE_APP_ID=

# Firebase Admin SDK (ตั้งค่าเชื่อมต่อ Firebase ฝั่งเซิร์ฟเวอร์)
FIREBASE_ADMIN_PROJECT_ID=
FIREBASE_ADMIN_CLIENT_EMAIL=
FIREBASE_ADMIN_PRIVATE_KEY=

# Vercel Blob Storage (ตั้งค่าที่เก็บรูปภาพ)
BLOB_READ_WRITE_TOKEN=
```
> ⚠️ **ข้อควรระวัง:** ไฟล์ `.env.local` เป็นความลับ ห้ามอัปโหลดขึ้น GitHub เด็ดขาด

### 4. รันโปรเจค (Start Development Server)
เมื่อติดตั้งและตั้งค่าเสร็จแล้ว ให้รันคำสั่ง:
```bash
bun run dev
```
จากนั้นเปิดเบราว์เซอร์ไปที่ **[http://localhost:3000](http://localhost:3000)** จะเห็นหน้าเว็บพร้อมใช้งาน! 🎉

## 📱 วิธีการตั้งค่าและ Build แอปพลิเคชันมือถือ (Android & iOS)

สำหรับโปรเจกต์ส่วนที่เป็นแอปพลิเคชันมือถือ (ใช้ Capacitor) ซึ่งจะแยกอยู่ในโปรเจกต์ `CMRU_Dorm_Mobile` สามารถทำตามขั้นตอนเพื่อรันและ Build ได้ดังนี้:

### 1. การเตรียมความพร้อม (Prerequisites)
- **สำหรับ Android:** ต้องติดตั้ง **Android Studio**
- **สำหรับ iOS:** ต้องติดตั้ง **Xcode** (ใช้งานได้เฉพาะบนระบบปฏิบัติการ macOS เท่านั้น)

### 2. การติดตั้งและการซิงค์ข้อมูลแอป
ให้เข้าไปที่โฟลเดอร์โปรเจกต์มือถือ (`CMRU_Dorm_Mobile`) จากนั้นรันคำสั่ง:
```bash
# ติดตั้งแพ็กเกจที่จำเป็น
npm install

# ซิงค์ข้อมูลหน้าเว็บล่าสุดไปยัง Android และ iOS
npm run sync
# (หรือใช้คำสั่ง: npx cap sync)
```

### 3. การรันและสร้างไฟล์ APK สำหรับ Android (ผ่าน Android Studio)
1. เปิด Android Studio โดยรันคำสั่งในโฟลเดอร์โปรเจกต์มือถือ:
   ```bash
   npm run build:android
   # (หรือใช้คำสั่ง: npx cap open android)
   ```
2. **การรัน (Run) เพื่อทดสอบ:** 
   - เมื่อโปรแกรมเปิดขึ้นมาและโหลดเสร็จ ให้เสียบสาย USB ต่อกับมือถือ (ที่เปิด Developer Mode แล้ว) หรือเลือกเปิดผ่าน Emulator 
   - กดปุ่ม **Run** (สัญลักษณ์ ▶️ สีเขียว) ที่แถบด้านบน
3. **การสร้างไฟล์ .apk (Build APK):**
   - ไปที่แถบเมนูด้านบน เลือก **Build** > **Build Bundle(s) / APK(s)** > **Build APK(s)**
   - รอจนระบบสร้างเสร็จ ไฟล์ APK จะถูกบันทึกไว้ที่: `android/app/build/outputs/apk/debug/app-debug.apk` 
   - (คุณสามารถนำไฟล์นี้ส่งเข้ามือถือเพื่อติดตั้งได้เลย)

### 4. การรันและ Build แอป iOS (ผ่าน Xcode - ต้องใช้ macOS)
1. เปิด Xcode โดยรันคำสั่งในโฟลเดอร์โปรเจกต์มือถือ:
   ```bash
   npm run build:ios
   # (หรือใช้คำสั่ง: npx cap open ios)
   ```
2. **การรัน (Run) เพื่อทดสอบ:**
   - เมื่อ Xcode เปิดขึ้นมา เลือก Simulator (เช่น iPhone 15) หรือเสียบไอโฟนเครื่องจริง
   - กดปุ่ม **Run** (สัญลักษณ์ ▶️) มุมซ้ายบน
3. **การ Build ส่ง App Store (Archive):**
   - ตั้งค่า Certificate และ Provisioning Profile ในหมวด **Signing & Capabilities** ให้เรียบร้อย
   - ไปที่แถบเมนู **Product** > **Archive** เพื่อเตรียมแอปส่งขึ้น TestFlight หรือ App Store
